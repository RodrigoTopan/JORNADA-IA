Teoria dos Grafos

A teoria dos grafos é um ramo da matemática que estuda as relações entre os objetos de um determinado conjunto. Dependendo da aplicação, arestas podem ou não ter direção, pode ser permitido ou não arestas ligarem um vértice a ele próprio e vértices e/ou arestas podem ter um peso (numérico) associado. Se as arestas têm uma direção associada (indicada por uma seta na representação gráfica) temos um dígrafo (grafo orientado).

Podemos dividir entre estruturas do tipo lista e do tipo matriz, mas em aplicações reais, a melhor estrutura é uma combinação de ambas. Estruturas do tipo lista são frequentemente usadas em grafos esparsos (grafos que possuem um pequeno número de arestas em relação ao número de vértices, oposto ao grafo denso, onde o número de arestas se aproxima do máximo de arestas possível) já que exigem menor uso da memória. Por outro lado, estruturas do tipo matriz fornecem um rápido acesso em algumas aplicações, mas podem consumir uma grande quantidade de memória.

Estruturas do tipo lista incluem a lista de adjacência que associa a cada vértice do grafo uma lista de todos os outros vértices com os quais ele tem uma aresta e a lista de incidência, que armazena para cada vértice uma lista de objetos que representam as arestas incidentes a esse vértice[7][8].

Estruturas do tipo matriz incluem a matriz de incidência, uma matriz de 0's e 1's com suas linhas representando vértices e suas colunas as arestas e a matriz de adjacência onde ambas linhas e colunas possuem vértices. Em ambos casos um 1 indica dois objetos adjacentes e 0 indica dois objetos não adjacentes.

APIs

O TensorFlow fornece várias APIs. A API de nível mais baixo se chama TensorFlow Core. Ela cai bem para cientistas de dados que precisam refinar muito seus dados.
As APIs de nível mais alto são boas para tarefas repetitivas, são mais fáceis de aprender e ajudam a gerenciar conjuntos de dados, estimadores e treinamentos.Ex: tf.estimator

Um Tensor

A principal estrutura do Tensorflow é um tensor. Basicamente, um tensor consiste em um conjunto de valores primitivos dentro de uma matriz de qualquer dimensão. O rank de tensores é o seu número de dimensões da matriz.
[1.,2.,3.] rank 1
[[1.,2.,3.], [4., 5., 6.]] rank 2
A estrutura do tensorflow core é comparada a um grafo de nós.

Declarando Constantes (NÓS)

Cada nó recebe zero ou mais tensores como entradas e produz um único tensor como saída Obs: um nó é uma constante(constant) e as constantes não precisam de valores de entrada

Imprimir valores dos nós(nodes)

Imprimir nós(nodes) não exibe os valores deles.
Para enxergar o valor dos nós temos que executar o print deles rodando dentro de uma sessão(Session)

Combinações e calculos

Podemos combinar tensores aos nós com operações (que também são nós)

Comandos Básicos


  • import tensorflow as tf
    • Importa o tensorflow para ser executado junto com o projeto em python
  • node1 = tf.constant(3.0, dtype=tf.float32)
    • Declara uma constante, que também é um nó(node)
  • print(node1, node2)
    • Exibe o nó e o seu tipo de dado com que trabalha na tela
  • sess = tf.Session()
    • Cria uma sessão e armazena em uma varíavel
  • print(sess.run([node1, node2]))
    • Exibe na tela os valores dentro de cada nó
