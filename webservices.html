WEB SERVICES

Explicando o AWS


Definição

São serviços disponibilizados via web que podem ser codificados em diversas linguagens e acessados sem dependência de similaridade com a plataforma solicitante. Você pode desenvolver um Web Service em Dot Net para disponibilizar algumas funcionalidades na Web e ter tais funcionalidades utilizadas por um outro sistema desenvolvido em Java.

Como funciona

Os Web Services utilizam o protocolo padrão da Internet (HTTP) tornando transparente a comunicação entre sistemas. Neste protocolo, os Web Services trafegam uma XML empacotada em um protocolo chamado SOAP (Simple Object Acces Protocol)

HTTP

A melhor forma de comunicação entre aplicações é sobre o protocolo HTTP, o protocolo HTTP é suportado por todos os navegadores de Internet e servidores. *O Hypertext Transfer Protocol (HTTP) - Protocolo de Transferência de Hipertexto - é um protocolo de comunicação (na camada de aplicação segundo o Modelo de referência OSI).

SOAP

SOAP - Simple Object Access Protocol, é um protocolo baseado em XML simples para permitir que os aplicativos a troca de informações sobre HTTP O SOAP é uma especificação que define o formato das mensagens XML. Uma mensagem SOAP nada mais é que um fragmento XML bem formado, encapsulado por um par de elementos SOAP.
